Tools Every Indie Developer Should Use

My actual toolkit — not a sponsored list.

There's no shortage of tools for developers. The problem isn't access — it's choosing what actually matters.

After building and shipping two apps solo, I've cut my stack down to tools that do one thing well: help me move faster without adding complexity. This isn't a "top 50 tools" list. It's what I actually use — and why each one earned its place.

The best stack is the one you actually use consistently. Not the most powerful. Not the most hyped. Just the one that lets you build and ship without friction.

Design

UI/UX Design

Figma

Still the best place to start. Clean interface, fast iteration, and it forces you to think through your product before writing code. If the design feels confusing in Figma, it will feel worse in code.

Visit Figma →

Development

Cross-Platform

Expo

If you're building cross-platform apps solo, this is hard to beat. Write one codebase, ship to both iOS and Android. You avoid native complexity while still shipping real apps. For a solo developer, speed is leverage.

Visit Expo →

Backend

Database & API

Supabase

If you need a backend, Supabase is one of the fastest ways to get there. Database, authentication, APIs, and storage — without setting everything up from scratch. The free tier is generous and the developer experience is genuinely good.

Visit Supabase →

Payments

Subscriptions

RevenueCat

Handles subscriptions across iOS and Android without the pain. Abstracts in-app purchases, subscription states, and cross-platform syncing. If you're charging users, this saves weeks of work and removes a major source of bugs.

Visit RevenueCat →

Version Control

Version Control

GitHub

Simple, reliable, and essential. Even solo, version control is non-negotiable. Commit history has saved me more than once — and it's the baseline for any serious project.

Visit GitHub →

AI Assistance

AI

Claude

Best used as a thinking partner — not a replacement for understanding. Useful for debugging, refactoring, and speeding up repetitive tasks. The key is knowing what not to trust blindly. AI works best when you already understand the problem.

Visit Claude →

Analytics

Analytics

Plausible Analytics

Privacy-friendly analytics without the complexity of traditional tools. No cookies, no personal data — just clean, simple numbers that tell you what you need to know. Fits naturally with a privacy-first philosophy.

Visit Plausible →

The Real Strategy

Tools don't make you productive — decisions do.

A smaller stack reduces context switching, makes debugging easier, and keeps your workflow clean. Most indie developers don't need more tools. They need fewer distractions.

  • Every tool you add is a tool you need to learn, maintain, and debug
  • The best tool for a job is often the one you already know
  • Switching tools mid-project is expensive — choose carefully upfront

The tools above made it into my stack because each one earned its place through consistent use — not because they were popular. Start with one or two, learn them properly, and expand only when you hit a clear limitation.
